### Architecture: HTLC-Based DvP Model 1

**HTLC (Hashed Time-Locked Contracts)**: Cryptographic atomicity across blockchain + banking

**DvP Model 1**: BIS-compliant simultaneous gross settlement

### HTLC Protocol (6-Step Process)

1. **Initialization**: Buyer generates secret S, hash H = SHA256(S)
2. **RUB Escrow**: ‚ÇΩ150K locked at bank with H, timeout T2=12h
3. **–¶–§–ê Escrow**: 10 –¶–§–ê locked in smart contract with H, timeout T1=24h
4. **Buyer Reveals**: Claims –¶–§–ê by revealing S on-chain
5. **Bank Payment**: Detects S, releases RUB via BESP (5-30 sec)
6. **Settlement**: Both legs complete, 3-7 min total

**Rollback**: If S not revealed, bank refunds @12h, blockchain @24h (asymmetric for safety)

### Smart Contract (Solidity)

```solidity
contract CFAHTLCEscrow {
    bytes32 public hashLock;
    uint256 public timelock;
    address public seller;
    address public buyer;
    IERC20 public cfaToken;
    uint256 public cfaAmount;
    bool public claimed;
    
    function withdraw(bytes32 _preimage) external {
        require(sha256(abi.encodePacked(_preimage)) == hashLock);
        require(block.timestamp <= timelock);
        claimed = true;
        cfaToken.transfer(buyer, cfaAmount);
    }
    
    function refund() external {
        require(block.timestamp > timelock && !claimed);
        cfaToken.transfer(seller, cfaAmount);
    }
}
```
